# 系统类型与关键拆解对象

拆解前先判断目标属于哪类系统。不同系统的关键对象完全不同——用养成系统的框架去拆新手引导会遗漏大量关键信息。

## 常见系统类型

| 系统类型 | 关键拆解对象 |
|----------|-------------|
| 活动系统 | 活动周期、入口、参与条件、目标、任务、奖励、付费点、刷新规则、结束规则 |
| 通行证系统 | 等级线、经验来源、免费奖励线、付费奖励线、任务结构、购买点、补领奖励 |
| 抽卡系统 | 卡池类型、货币、概率、保底、重复转化、UP机制、付费路径、展示反馈 |
| 商店系统 | 商品分类、价格、刷新、限购、折扣、推荐逻辑、付费货币、资源缺口 |
| 养成系统 | 养成对象、材料来源、成长阶段、消耗曲线、属性反馈、卡点和长期目标 |
| 关卡系统 | 关卡目标、失败条件、难度曲线、题眼、奖励、消耗、重复挑战机制 |
| 战斗系统 | 操作方式、角色能力、敌人行为、胜负条件、数值反馈、策略空间 |
| 合成/收集系统 | 操作空间、产出来源、物品链、目标需求、空间压力、图鉴、背包、商店补给 |
| 新手引导 | 引导阶段、教学目标、强制步骤、奖励、解锁节奏、跳出点 |
| 广告变现 | 触发时机、奖励内容、观看频次、冷却、收益翻倍、失败补救、付费替代 |
| 社交系统 | 好友关系、互动方式、排行榜、公会/社团、协作玩法、社交奖励 |
| 签到/日常系统 | 签到周期、奖励递增、补签机制、日常任务、周常月常、重置规则 |
| 赛季/排名系统 | 赛季周期、段位结构、匹配规则、奖励分层、重置机制、保护机制 |

## 识别方法

如果目标不完全匹配上述类型：

1. 先看系统的**核心动作**——玩家在这个系统里反复做什么？
2. 再看**驱动循环**——是什么让玩家持续回来？
3. 最后看**商业化角色**——这个系统在变现链条中处于什么位置？

根据这三点自行定义类型，并列出对应的关键拆解对象。

## 混合系统

有些系统是混合型的（比如"通行证+活动"、"商店+养成"），这种情况下：
- 分别列出两种类型的关键对象
- 重点关注两个系统之间的**连接点**——它们如何互相驱动
- 标记哪些对象属于哪个子系统
