# Game Teardown Skill

## 游戏系统/玩法/功能 竞品拆解技能

一个 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) Skill，用于结构化拆解竞品游戏的系统、玩法和功能，输出可供策划直接参考的拆解报告。

### 它能做什么

给出一款游戏和一个系统（比如"帮我拆解《xxx》的xxx功能"），它会：

1. **主动收集你的项目背景** — 确认你的项目类型、阶段、商业化方式，确保迁移建议不是纸上谈兵
2. **联网搜索公开资料** — 自动搜索官方公告、攻略、Wiki、玩家评论、视频等，补齐信息缺口
3. **事实优先，标记可信度** — 区分"截图确认"、"玩家攻略"、"资料推断"、"待验证"，不把猜测写成结论
4. **输出 17 章节结构化报告** — 从系统结构、核心流程、规则拆解、商业化分析到迁移方案，一份报告覆盖完整

### 报告结构

| 阶段 | 章节 | 内容 |
|------|------|------|
| 事实整理 | 1-4 | 系统类型识别、截图/资料事实提取、联网搜索补齐、可信度标记 |
| 结构拆解 | 5-7 | 子模块构成、核心玩法流程、14项规则维度 |
| 深度分析 | 8-12 | 资源与商业化、UI交互路径、体验设计、设计亮点、潜在风险 |
| 迁移建议 | 13-16 | 可借鉴点、不建议照搬、适配方案、策划验证清单 |
| 收尾 | 17 | 报告优化建议（告诉你补充什么素材可以升级哪些章节） |

### 安装

将本仓库克隆到 Claude Code 的 skills 目录：

```bash
cd ~/.claude/skills
git clone https://github.com/JunChowABC/game-teardown-skill.git game-teardown
```

安装后，在 Claude Code 中输入 `/game-teardown` 即可触发，或在对话中自然地说：

- "帮我拆解XX游戏的XX系统"
- "分析一下XX的商店设计"
- "参考XX的通行证怎么做的"

### 文件结构

```
game-teardown/
├── SKILL.md                          # 核心技能文件（触发条件、执行原则、工作流程）
├── references/
│   ├── system-types.md               # 13种系统类型与对应关键拆解对象
│   ├── fact-extraction.md            # 事实提取模板、可信度标记、来源定义
│   └── rule-dimensions.md            # 14项规则维度详解
└── templates/
    └── report.md                     # 完整报告输出模板
```

### 设计原则

- **先事实后分析** — 先整理能确认的信息，再做设计判断
- **能量化的必须量化** — "很多"和"挺贵"不是策划能用的信息
- **推断不是事实** — 所有信息标记来源和可信度
- **必须落到可用结论** — 不只说竞品怎么做，还要说为什么、是否适合迁移、怎么改

### 支持的系统类型

活动系统、通行证、抽卡、商店、养成、关卡、战斗、合成/收集、新手引导、广告变现、社交、签到/日常、赛季/排名 — 以及任何自定义类型。

### 使用示例

**输入：**
> 帮我拆解《xxx》的xxx功能

**Skill 会：**
1. 先问你的项目类型、阶段、商业化方式
2. 联网搜索剑零AI托管的公开资料
3. 整理事实 → 拆解结构 → 分析规则 → 评估商业化
4. 输出针对你项目的迁移方案和验证清单
5. 告诉你补充哪些截图/数据可以升级报告

### 适配场景

- 竞品调研立项
- 版本迭代前的竞品对标
- 新系统设计前的参考分析
- 策划培训和知识沉淀
